PROGRAMA.SH

PROGRAMA wifi.sh para rtl8811au
Reinstalar despues de actualizar la version del kernel
Ejecutar en la terminal sudo sh wifi.sh
============================================
#!/bin/bash
#!/bin/bash
clear
while :
do
echo " Escoja na opcion "
echo "1. Ir a carpeta rtl8812AU"
echo "2. Reinstalar driver"
echo "3. modprobe"
echo "4. Reiniciar Tarjeta Red"
echo "5. Salir"
echo -n "Seleccione una opcion [1 - 5]"
read opcion
case $opcion in
1) echo "Carpeta de drivers";
cd rtl8812AU_8821AU_linux && ls -l;;
2) echo "Reinstalar";
sudo make && sudo make install;;
3) echo "SUDO Modprobe";
sudo modprobe rtl8812au;;
4) echo "Reiniciando RED";
sudo /etc/init.d/networking restart;;
5) echo "agur";
exit 1;;
*) echo "$opc no es una opcion válida.";
echo "Presiona una tecla para continuar...";
read foo;;
esac
done

    UPGRADE COPIANDO FILES


    Replace your core.gz and vmlinuz files.
    The 32 bit versions are here (core.gz and vmlinuz):
    http://tinycorelinux.net/11.x/x86/release/distribution_files/
    The 64 bit versions are here (corepure64.gz and vmlinuz64):
    http://tinycorelinux.net/11.x/x86_64/release/distribution_files/

    Funcionamiento tiny core linux

    Arquitectura :
    Consta de dos archivos:

    vmlinuz

    y

    core.gz.

    Se cargan en la memoria. Aplicaciones añadidas:

    /mnt/sda1/tce/optional

    Se cargaran si figuran en le fichero

    /mnt/sda1/tce/onboot.lst


    Las configuraciones que hace el usuario se guardan en el fichero comprimido

    /mnt/sda1/tce/mydata.tgz


    En el arranque se descomprime este fichero en los directorios

    $HOME - /opt

    y en los que figuren en el archivo

    /opt/.filetool.lst


    Despues del boot se ejecuta el scrip

    /opt/bootlocal.sh

    si queremos añadir comandos concretos en el arranque los debemos incluir en este archivo
    El script

    filetool.sh

    TinyCore Linux proporciona la función de copia de seguridad que soporta la persistencia . El script hace una copia de seguridad de archivos creados en el directorio

    /opt

    y directorios

    $ HOME

    El script también copia de seguridad de los directorios que contienen los archivos de configuración asociados con software como Quagga red, si esos directorios se enumeran en el archivo

    /opt/.filetool.lst


    Cambios en la configuración de la red hacen uso de los comandos de Linux como el comando ip no se puede guardar.
    Sin embargo, podemos agregar estos comandos a la secuencia de comandos shell

    /opt/bootlocal.sh

    , que se ejecuta en el arranque.
    Una vez que haya creado las configuraciones de red que desea guardar, editando el script

    /opt/bootlocal.sh

    y

    /

    o guardar los archivos de configuración o Quagga Open vSwitch, una copia de seguridad de la configuración con el script

    filetool.sh

    de la siguiente manera:

    $ filetool.sh -b


    Para apagar un sistema TinyCore Linux, utilice el comando:

    $ sudo poweroff


    Para reiniciar un sistema TinyCore Linux, utilice el comando:

    $ sudo reboot
